These guidelines are presented in the hope of making the skin plugin easier to use and minimising potential problems which could occur later. Any user of the skin plugin is invited to contribute to this section for the benefit of all.

A debug log file is created while using the skin plugin called c:\skinDebug.log.

If you have any problems whilst using this plugin, please send me the following: the file you were working on, the debug log, and a short message describing what you were doing or intended to do.


Before Using The Skin PlugIn

If after you have copied the plugin into you MAX plugins directory and you find it isn't available in the Create / SpaceWarps menu you probably need to copy Msvcrtd.dll into your Windows / System directory. If the file already exists, don't bother overwriting the file. This file is used internally for debug logging.


General Use

Always make sure that you are using the latest version of the plugin before reporting any problems.


Preparing An Object For Skinning

Although it is perfectly acceptable to add a skin to any object, it is recommended the user convert the object to an editable mesh before binding.

It is also recommended that before adding a skin to an object that the objects "Pivot" transform is reset to identity. Use the Pivot plugin for this purpose.

Assigning Vertices

Always perform the vertex assignments at frame zero. This is because the assignment is made relative to the bones reference frame which is frame zero of the animation.


Modifying The Mesh After Vertex Assignment

Modifications to the mesh after assignment should be picked up automatically by the modifier and the assignments updated. If this doesn't happen please let me know.

There seems to be a problem with MAX temporarily losing internal references inside the skin modifier if you make any mesh modifications. One way to prevent this is before moving back to the skin modifier in the modify stack, select ALL vertices in the Edit Mesh modifier.
